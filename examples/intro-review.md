# Introduction Review Example

## User Prompt

```text
Use $paper-mentor to review my introduction.

Field: machine learning for education
Target venue: general applied AI conference
Claimed contribution: a lightweight feedback model for short student answers
Concern: I am not sure whether the novelty is clear.

Introduction:
[paste introduction here]
```

## Expected Response Shape

```text
Summary judgment:
The introduction should be evaluated for whether it makes the educational problem concrete, distinguishes the proposed model from existing feedback systems, and links the method to evidence.

Advisor:
- Is the audience-specific problem clear?
- Is the gap more than "existing methods are bad"?
- Is the contribution stated as a testable claim?

Reviewer 2:
- The likely rejection risk is insufficient novelty unless the introduction names the closest prior systems and the exact difference.

Method Auditor:
- The introduction should preview what evidence supports the claimed lightweight design.

Writing Coach:
- The opening should move from real educational setting to technical gap to contribution.

Next three actions:
1. State the closest prior approach.
2. Define what "lightweight" means operationally.
3. Preview the evaluation setup and strongest result.
```
