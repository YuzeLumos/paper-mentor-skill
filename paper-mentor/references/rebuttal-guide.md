# Rebuttal Guide

Use this reference when the user has reviewer comments, meta-review feedback, revision requests, or wants to prepare a response letter.

## Principles

- Answer the concern behind the comment, not just the literal wording.
- Be respectful, concrete, and evidence-based.
- Separate manuscript changes from explanations.
- Admit limitations when the reviewer is right.
- Do not promise new experiments or analyses unless the user can actually perform them.
- Do not invent results, citations, reviewer intent, or manuscript changes.

## Comment Triage

Classify each comment:

- Clarification: wording, definition, structure, missing explanation
- Evidence: missing experiment, analysis, baseline, citation, or example
- Scope: overclaiming, unsupported generalization, missing limitation
- Method: design, metric, dataset, reproducibility, validity
- Positioning: novelty, related work, contribution framing
- Presentation: figure, table, formatting, readability

## Response Strategy

1. Restate the concern neutrally.
2. Decide whether to agree, partially agree, or respectfully clarify.
3. Name the manuscript change.
4. Connect the change to evidence.
5. Avoid broad claims that the revision cannot support.

## Draft Response Pattern

```text
We thank the reviewer for raising this concern. We agree that [valid point]. In the revised manuscript, we [specific change]. This addresses the concern by [evidence or clarification]. We also [additional concrete change if applicable]. We have revised [section/table/figure] accordingly.
```

## Output Template

```text
Reviewer concern:

Category:
Severity:

Manuscript change needed:
Evidence needed:

```

## Tone Checks

Avoid:

- "The reviewer misunderstood" as a first move
- defensive explanations without manuscript changes
- claims that go beyond available evidence
- long responses that hide the answer

Prefer:

- "We have clarified..."
- "We now report..."
- "We added an analysis..."
- "We acknowledge this limitation..."
- "We revised the text to avoid overclaiming..."
